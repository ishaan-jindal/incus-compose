# Plan an issue

Initialize context as in `.zed/prompts/hello.md`.

Then plan issue `#$ARGUMENTS` without implementing code yet.

Use `glab` or `gh issue` only if available and appropriate. If the issue content cannot be read from local context or tools, ask for the issue text instead of guessing.

The plan should include:

- relevant files and packages to inspect
- expected implementation approach
- tests or `just` commands to run
- risks, unknowns, or questions
